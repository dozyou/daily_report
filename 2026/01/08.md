## わかったこと
- `git rebase <ブランチ名>`：枝分かれしたコミットを直線的になるように統合する。
- `git pull --rebase <リモート名> <ブランチ名>`：マージコミットを残さずGitHubの内容をローカルに持ってくる。
- `git commit --amend`：直前のコミットをやり直す。
- `git rebase -i`：コミットの削除と修正を行う。
  - `HEAD~<遡るコミット数>`：`-i`の後に続けて記述。HEADを基点にして数値分の親コミットまで指定する。
  - `HEAD^`：`-i`の後に続けて記述。マージした場合の2番目の親を指定する。
  - 上記コマンドを入力するとエディタが立ち上がる。エディタでpick⇨editに書き換えることで、対象のコミットを修正する。pick⇨squashに書き換えることでsquashに書き換えたコミットを1つにまとめる。
- `git tag`：タグの一覧を表示する。
- `git tag -l "検索条件"`：検索条件で部分一致するタグを一覧で表示する。
- `git tag -a <タグ名> -m "メッセージ"`：注釈付きタグ(名前/コメント/署名付き)を作成する。
- `git tag <タグ名>`：軽量版タグを作成する。
- `git tag <タグ名> <コミット名>`：過去のコミットに対してタグをつける。
- `git show <タグ名>`：タグのデータと関連づけられた情報を表示する。
- `git push <リモート名> <タグ名>`：タグをリモートリポジトリにプッシュする。
- `git push <リモート名> --tags`：リモートリポジトリにないタグを全てプッシュする。
- `git stash`または`git stash save`：コミットしたくはないが、急遽別ブランチで作業する必要に迫られたときに作業を一時避難する。
- `git stash list`：stashに避難した作業を確認する。
- `git stash apply`：最新の作業を復元する。
- `git stash apply --index`：ステージの状況も合わせて復元する。
- `git stash apply <スタッシュ名>`：特定の作業(スタッシュ名)を復元する。
- `git stash drop`：stashに避難した最新の作業を削除する。
- `git stash drop <スタッシュ名>`：指定した作業(スタッシュ名)を削除する。
- `git stash clear`：スタッシュにある全ての作業を削除する。
