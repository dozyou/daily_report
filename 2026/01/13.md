## わかったこと
- git branch -f <ブランチ名> <コミット>：ブランチを特定のコミットに移動させる。
- git cherry-pick <コミット1> <コミット2> <…>：今いるブランチにコミット1、コミット2、…の順にコミットを持ってくる。
- git rebase <ブランチ1> <ブランチ2>：まずブランチ1に切り替え(checkout)、その後ブランチ1をブランチ2の先端に移動させる(rebase)。
- git describe：今いるコミットがどのタグから派生し、どれくらい離れているかを表示する。
  - 実行結果がv1.0.2-4-g9c1b234の場合、v1.0.2が最も近いタグ名であり、そのタグから現在までに4コミットが追加されている。現在のコミットのハッシュ(冒頭)はg9c1b234である。
- git checkout -b <ブランチ名> origin/main(追跡対象のリモートブランチ)：origin/mainと同じ場所(コミット)に新しくブランチを作成し、切り替える。その上で、作成したブランチの上流としてorigin/mainを設定する。
- git branch -u origin/main(追跡対象のリモートブランチ) <ブランチ名>：すでに存在するブランチに対して、追跡対象のブランチを指定する。
