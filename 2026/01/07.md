## わかったこと
- `git pull <リモート名> <ブランチ名>`：<リモート名>の<ブランチ名>をローカルリポジトリに取得した後、ワークツリーにその情報を加える。
- `git pull`の動作は特殊。今自分がいるブランチにリモートからとってきた情報をマージしてしまう。
- `git remote show <リモート名>`：FetchとPushのURL/リモートブランチ/git pushの挙動など詳しい情報を見ることができる。
- `git remote rename <旧リモート名> <新リモート名>`：<旧リモート名>を<新リモート名>に変更する。
- `git remote rm <リモート名>`：<リモート名>を削除する。
- `git branch <ブランチ名>`：<ブランチ名>を新規作成する。※ブランチを作成するだけでブランチの切り替えまでは行わない。
- `git branch`：ローカルリポジトリのブランチの一覧を表示する。
- `git branch -a`：リモートリポジトリも含めた全てのブランチを一覧表示する。
- `git checkout <既存ブランチ名>`：<既存ブランチ名>にブランチを切り替える。
- `git checkout -b <新ブランチ名>`：<新ブランチ名>を新規作成して切り替える。
- `git merge <ブランチ名>`または`git merge <リモート名/ブランチ名>`：現在のブランチに<ブランチ名>の変更差分をマージする。
- `git branch -m <ブランチ名>`：ブランチ名を変更する。
- `git branch -d <ブランチ名>`：ブランチ名を削除する(※masterにマージされていない変更がある場合は削除しない)。
- `git branch -D <ブランチ名>`：ブランチを強制削除する。
