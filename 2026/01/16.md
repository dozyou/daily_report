## わかったこと
- `git push <リモート名> <ブランチ名>`：ローカルリポジトリのブランチ名をリモートリポジトリのブランチ名が一致するブランチにpushする。
- `git push <リモート名> <特定コミット>:<ブランチ名>`：特定コミットをリモートリポジトリのブランチ名にpushする。この時、ブランチ名はローカルリポジトリのブランチ名と同一でなくても良い。また、ブランチ名に存在しないブランチを設定すると、その名前で新規ブランチを作成する。
- GitHubでリポジトリを作成後、ローカルリポジトリと繋ぎ込みを行う手順
  - `git init`:今いるディレクトリをGitリポジトリに変換する。
  - `git remote add origin <GitHubのURL>`：GitHubの指定したリポジトリとローカルリポジトリを繋ぎこむ。
  - `git pull origin main`：リモートリポジトリの中身をローカルに入れる。
- 上記手順は`git clone <GitHubのURL>`で代替可能。
